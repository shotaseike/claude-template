Update and maintain project documentation (CLAUDE.md, README.md, and other docs).
Check what needs to be updated in the project documentation, then guide the user through the updates.

## Step 1: Verify documentation exists

Check if `CLAUDE.md` exists in the project root:
- If **exists**: Read it, identify outdated/incomplete sections
- If **missing**: Suggest running `/init-project` first to generate it

If `README.md` exists, also read it to check consistency with CLAUDE.md.

---

## Step 2: Identify documentation gaps

For each documentation file, identify:

### CLAUDE.md sections to check
1. **Project Overview** — Is the project description clear? Is framework/language still accurate?
2. **Environment Setup** — Are all required environment variables documented? Any deprecated or missing ones?
3. **Common Commands** — Do the commands match current project structure? Are there new build/test/run commands?
4. **Architecture** — Is the module list and data flow still accurate?
5. **Claude Code Configuration** — Do the installed rules/skills/agents still match what `/init-project` installed?

### README.md
- Does it describe the project purpose, features, and setup?
- Are all links (to CLAUDE.md, other docs) still valid?
- Is the getting-started guide up-to-date?

---

## Step 3: Report findings to user

For each issue found, explain:
- **What's outdated**: quote the stale text
- **Why it matters**: how this affects clarity or usability
- **Suggested fix**: a brief rewrite or clarification

Format as a checklist:
```
- [ ] CLAUDE.md § Environment Setup: add DATABASE_URL env var
- [ ] CLAUDE.md § Common Commands: update test command to 'pytest'
- [ ] README.md: update Getting Started link
```

Ask the user:
> これらのドキュメント更新をお手伝いしましょうか？優先順位や変更内容について教えてください。

---

## Step 4: Update documentation (interactive)

Once the user approves, guide them through each update:

1. Ask for clarification on each point
2. Draft the new content
3. Show the before/after diff
4. Confirm before writing

Example:
```
### Environment Setup に DATABASE_URL を追加

【現在】
<!-- TODO: Add required environment variables and setup steps -->

【提案】
## Environment Setup

Required environment variables:
- `DATABASE_URL` — PostgreSQL connection string (e.g. `postgres://user:pass@localhost:5432/mydb`)
- `SECRET_KEY` — API secret for authentication

Optional:
- `DEBUG` — Set to `true` for verbose logging (default: `false`)

これで大丈夫ですか？
```

## Tips for maintaining documentation

**CLAUDE.md is sacred** — it's the main reference for you and Claude. Keep it:
- ✅ Accurate (reflect current code/setup)
- ✅ Concise (< 200 lines total; use links to longer docs for details)
- ✅ Up-to-date (review whenever architecture changes)

**How often to run /docs**:
- After major architecture changes
- When adding new dependencies or environment variables
- When onboarding a new team member (to spot clarity gaps)
- Before deploying to production

**Links to COMPONENTS.md**:
- CLAUDE.md § Claude Code Configuration should reference [COMPONENTS.md](COMPONENTS.md) for details on rules/skills/agents
- COMPONENTS.md is auto-generated by `/init-project` and `/update-claude-config`; no need to manually maintain
